# ROADMAP

## Phase 0 - Bootstrap
- Confirm the repository structure.
- Seed `.planning` state files.
- Add a reusable session hygiene script.

## Phase 1 - Next Requested Task
- Wait for the concrete feature or bug request.
- Map the relevant code paths.
- Make the smallest safe change.

## Phase 2 - Verification
- Run targeted checks for the touched code paths.
- Capture any residual risk or follow-up work.

## Phase 3 - Handoff
- Update the planning checkpoint.
- Summarize what changed and what remains.
