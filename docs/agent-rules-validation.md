# Agent Validation Rules

## 자동수정
- 사용자가 오류/기대 결과를 지적하면, 금융 감지/저장 로직 수정의 경우 먼저 `docs/issue-history-volatile/`에 이슈를 기록한다.
- 기록에는 원문, 문제, 기대 결과를 포함한다.
- 금융 원문 보관 데이터와 이슈 기록은 같은 폴더에 두지 않는다.

## 전수 점검
- 기능/로직 수정 후에는 수정 파일만 보지 말고 아래를 함께 본다.
  - 진입점(UI 이벤트)
  - 데이터 조회 소스
  - 저장 경로
  - 후속 화면(상세/더보기)
  - 예외/폴백 경로
- 하나라도 빠지면 완료 처리하지 않는다.

## 테스트/검증
- 타입체크, 빌드, 테스트, 검증 명령이 하나라도 실패하면 완료 보고를 하지 않는다.
- 실패 시 원인을 수정한 뒤 같은 검증을 다시 실행한다.
- 금융 감지부터 저장까지 수정했다면 관련 회귀를 다시 돌린다.

## 금융 회귀
- 금융 감지/저장 로직 변경 시 다양한 표본을 포함해 확인한다.
  - 카드 승인
  - 체크카드
  - 계좌이체
  - 입금/출금
  - 취소
  - 간편결제
  - 할부
  - 실패 이력 케이스
- 관련 표본 중 하나라도 기대와 다르면 완료 보고를 하지 않는다.

## 완료 보고
- 완료 선언 전에는 사용자의 최근 요청을 다시 대조한다.
- 결과, 검증, 사용 방법, 남은 실제 리스크만 짧게 정리한다.
- 빌드/설치/실행을 직접 했으면 최종 보고에 굵게 표시한다.
